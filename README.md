# Анализ повторных заказов в сервисе Яндекс Афиша

Учебный аналитический проект по данным сервиса Яндекс Афиша. Цель работы — понять, какие признаки первого заказа и поведения пользователя связаны с вероятностью повторной покупки, и на основе этого сформулировать практические рекомендации для маркетинга.

Основной ноутбук проекта находится в `yandex_afisha_analysis.ipynb`. В нём сохранены результаты вычислений и графики.

## Что сделано

- выгружены данные из PostgreSQL и собран единый датасет по заказам за 2024 год;
- выполнена предобработка: перевод выручки из тенге в рубли, анализ пропусков, типов данных и выбросов;
- построен пользовательский профиль с агрегированными признаками;
- проведён исследовательский анализ факторов, связанных с возвратом пользователей;
- подготовлены выводы и рекомендации для маркетинговой команды.

## Коротко о результатах

- В анализе использованы данные по **290 611 заказам** и **21 769 пользователям**.
- Доля пользователей с повторными покупками составила **61,8%**, а пользователей с **5+ заказами** — **29,1%**.
- Пользователи из сегмента **2–3 билета в заказе** показали лучшее удержание: **73,9%** против среднего уровня **61,8%** по выборке.
- В активных регионах доля повторных покупок выше, чем в менее активных: **59,5%** против **52,7%**.
- Пользователи с **5+ заказами** возвращаются значительно быстрее: в среднем через **9,6 дня** против **21,4 дня** у группы с 2–4 заказами.
- Наиболее полезными признаками для будущей ML-модели оказались **регион** и **билетный оператор** первого заказа.

## Практическая польза

- Выделен приоритетный сегмент для удержания: пользователи, покупающие **2–3 билета**, уже показывают высокую склонность к повторным заказам и подходят для персональных рекомендаций и механик наподобие «пригласи друга».
- Найдена группа риска: **38,2%** пользователей сделали только один заказ. Для них имеет смысл тестировать CRM-коммуникации в первые недели после покупки.
- Показано, что средняя выручка с заказа почти не влияет на возврат, поэтому в маркетинговых сценариях важнее опираться не на чек, а на поведенческие признаки пользователя.
- Результаты анализа можно использовать как основу для ML-модели прогнозирования повторных заказов и для сегментации пользователей в CRM.

## Стек

`Python`, `pandas`, `numpy`, `matplotlib`, `seaborn`, `SQLAlchemy`, `phik`, `Jupyter Notebook`

## Структура репозитория

- `yandex_afisha_analysis.ipynb` — основной ноутбук проекта;
- `final_tickets_tenge_df.csv` — данные по курсу тенге к рублю для пересчёта выручки;
- `requirements.txt` — список библиотек для запуска проекта.

## Как открыть проект

1. Установить зависимости:
   `pip install -r requirements.txt`
2. Открыть `yandex_afisha_analysis.ipynb` в Jupyter Notebook или JupyterLab.

Ноутбук уже сохранён с выводами, поэтому его можно просто открыть на GitHub и посмотреть как готовый кейс. Для повторного запуска SQL-части нужно подставить свой учебный пароль к базе в ячейке с `db_config`.
